One-time setup

Do this once when you first start
Change the owner password

The default password is ichigo-admin — change it before sharing the app with anyone.

Input → Owner password card → type a new password → Change

Add your employees

Each employee needs a name and a PIN (4+ digits) to log in.

Input → Employees card → fill in name + PIN → Add employee
Share each person's name + PIN with them directly. They can change their PIN after logging in.

Add your clients

Clients are the restaurants and cafes you sell to.

Input → Clients card → enter client name → Add

Set pay rates

These drive the salary calculation. Check they're correct.

Input → Pay rate card
• Piece rate — dollars earned per daifuku produced (default $2.00)
• Gusto base rate — the hourly rate used to convert total pay into Gusto hours (default $20.00)

Set target productivity (optional)

Used to suggest how many people to assign based on the day's order volume.

Input → Target productivity card → enter pieces per person-hour → Save

{# ── Section 2: Open the month ───────────────────────────────────────────── #}
2

Open the month

Do this ~3–4 weeks before the month starts
Generate shift dates

This creates the Mon/Tue/Wed/Fri dates for the chosen month so employees can submit availability.

Input → Open a month card → type the month (e.g. 2026-08) → Generate shifts
Safe to run again — it won't create duplicates.

Tell your team to submit availability

Employees log in and go to My availability to mark which shifts they can work, including partial time windows if needed.

Give them a deadline — at least 1 week before the month starts is recommended.

{# ── Section 3: Approve availability ─────────────────────────────────────── #}
3

Approve availability

After employees submit, before you assign
Review submissions

Each submission shows the employee name, which shift, and their available time window.

Approvals → review each row → click Approve or Reject
Only approved submissions appear as options when you assign people to shifts.

Check coverage

Make sure enough people are available for each shift (min/max targets: Mon 3, Tue 4–5, Wed 4–5, Fri 5–7).

If a shift looks short, reach out to employees before assigning.

{# ── Section 4: Enter orders ─────────────────────────────────────────────── #}
4

Enter client orders

Before assigning staff — orders drive the suggested team size
Add orders per production day

Enter how many of each flavor (Original / Matcha / Hojicha / Other) each client ordered for that day.

Two ways to get there:
• Orders → pick the month → click Edit orders on any row
• Input → upcoming shifts table → Edit orders button on any day

Edit or remove orders

On any day's order page, quantities are editable inline — just change the numbers and hit Save. Use the red ✕ to delete an order.

{# ── Section 5: Assign people ────────────────────────────────────────────── #}
5

Assign people to shifts

Once orders are entered and availability is approved
Open the schedule

Schedule → pick the month → see all shift dates with staffing status

Each row shows how many people are assigned vs. needed, and total pieces ordered.

Auto-assign (quick option)

The app picks people automatically to keep the average headcount within the min/max band.

On the Schedule page: ⚡ Auto-assign empty shifts button fills every unstaffed shift at once.
Or open a specific shift → ⚡ Auto-assign from approved availability for just that day.

Manual assignment

Use this to override auto-assign or fine-tune individual shifts.

Schedule → click a shift date → People section:
• Tick the checkbox next to each person to include them
• Adjust their start/end window if needed (e.g. someone can only come for part of the shift)
• Tick Manager 👑 for whoever is leading that shift (+5% pay)
• Click Save assignment

The Coverage bar chart shows average headcount across the shift — amber = below minimum, red = above maximum.

Check the calendar

Calendar → verify the month looks right before sharing with your team

Employees can see the calendar anytime — their own shifts are highlighted.

{# ── Section 6: Confirm hours ────────────────────────────────────────────── #}
6

Confirm actual hours worked

After each production day
Go to the shift's detail page

Schedule → click the date → scroll to "Confirm actual hours worked" at the bottom

Review and adjust times

Times are pre-filled automatically:

• Manager 👑 — start time is set 15 minutes before the official shift start
• Everyone else — start/end match their assigned window

Change any time if someone arrived late, left early, or stayed longer. The hours calculate live as you type.

Save

Click Save actual hours → status changes to ✓ confirmed
These confirmed times are then used in the salary calculation.

{# ── Section 7: Salary & Gusto ───────────────────────────────────────────── #}
7

Salary & Gusto

At the end of the pay period
Add any pop-up hours

For markets, events, or any work outside the regular production schedule.

Salary → Add pop-up / extra hours form:
Select the employee, date, start time, end time, hourly rate, and any transport cost (bridge toll, BART, etc.).
Leave the rate blank to use the default Gusto rate (${{ gusto_rate }}).

Set the date range and calculate

Salary → set From and To dates for the pay period → Calculate

How the hourly rate is calculated per shift:
rate = (${{ piece_rate }} × total daifuku produced) ÷ total person-hours on that shift
Manager pay = rate × hours × 1.05 (5% bonus). All other workers = rate × hours.

Read the Gusto input table

The highlighted Gusto hours column is the single number to enter per person in Gusto at ${{ gusto_rate }}/hr.

Formula: Gusto hours = (total wage pay + transport costs) ÷ ${{ gusto_rate }}
Transport is folded in so you enter one number per person — no separate line items needed.

Shifts showing ⚠ unconfirmed means actual hours haven't been saved yet on that shift's detail page — go confirm them first for accurate numbers.

Enter hours into Gusto

In Gusto, run payroll and enter each person's Gusto hours at the ${{ gusto_rate }}/hr base rate.
That's it — the app has done all the math.
